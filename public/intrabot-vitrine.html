Pipeline RAG

De la question à la réponse sourcée

Chaque requête suit un pipeline déterministe en cinq étapes; aucune réponse n'est inventée, tout est ancré sur les documents indexés.

⌕
Question
L'utilisateur pose une question en langage naturel
⟳
Embedding
La question est vectorisée par Cohere embed
◎
Retrieval
ChromaDB retourne les chunks les plus proches
⊞
Augmentation
Le prompt est allimenté avec les passages sourcés
✦
Génération
Le LLM produit une réponse avec citations
Fonctionnalités

Tout ce dont une équipe a besoin

IntraBot couvre le cycle complet, de l'ingestion des documents jusqu'au suivi de la qualité des réponses, avec des garde-fous pensés pour un usage professionnel.

⌕
Recherche documentaire RAG
Questions en langage naturel sur toute la documentation interne. Les réponses citent leurs sources avec un score de pertinence.
search · retrieval
◈
Contrôle d'accès par rôle
5 profils (Collaborateur, Ingénieur, Manager, RH, Admin) avec accès différencié. Un RH ne voit que les documents RH.
RBAC · sécurité
↑
Gestion documentaire admin
Upload, réindexation et suppression depuis l'interface. Ingestion incrémentale, seuls les fichiers nouveaux sont retraités.
ingestion · admin
✓
Workflow de validation
Les utilisateurs soumettent des documents selon leur profil. L'admin examine et approuve avant intégration au chatbot.
workflow · approbation
◑
Feedback et suivi qualité
Chaque réponse est notable (👍/👎). Une page Activité affiche le taux de satisfaction et les retours par rôle pour l'admin.
analytics · feedback
⊘
Garde-fous anti-hallucination
Le LLM répond uniquement à partir des documents indexés. Les questions hors sujet reçoivent un refus explicite, sans invention.
guardrails · fiabilité
Architecture

Trois couches, deux microservices, un gateway

Une architecture hexagonale découplée : chaque composant peut évoluer indépendamment, et le passage par l'orchestrateur se résume à changer deux variables d'environnement.
